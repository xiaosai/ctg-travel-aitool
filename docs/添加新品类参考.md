## 二、添加新品类的步骤

### 步骤 1：梳理后端 API 接口

**目标**：整理出新品类相关的所有 API 接口清单。

**输出格式**：

| 序号 | method | 描述 | category | subCategory | action | 必填参数 |
|------|--------|------|----------|-------------|--------|----------|

**参考现有分类**：

| 分类 | 说明 | 示例接口 |
|------|------|----------|
| RESOURCE/COMMON | 公共资源 | cityList（城市列表） |
| RESOURCE/{CATEGORY} | 品类资源 | search、detail |
| PASSENGER/COMMON | 乘客/入住人 | getPassengerList、savePassenger |
| ORDER/{CATEGORY} | 品类订单 | createOrder、cancelOrder、refund |
| ORDER/COMMON | 通用订单 | orderHistory、orderDetail、getOrderStatus |

**执行方式**：
1. 用户提供后端接口文档/代码/Swagger 地址
2. 或用户提供接口清单，我协助整理格式

### 步骤 2：编写 `api/{category}.json`

**文件位置**：`/Users/zhangsaiyong/guolv/projects/ctg-travel-aitool/api/{category}.json`

**格式规范**：

```json
[
  {
    "method": "cityList",
    "description": "城市列表",
    "category": "RESOURCE",
    "subCategory": "COMMON",
    "action": "CITY",
    "parameters": {
      "type": "object",
      "properties": {
        "domesticType": {
          "type": "integer",
          "description": "国内/国际类型 1：国内 2：国际"
        },
        "resourceType": {
          "type": "integer",
          "description": "资源类型 (飞机=0, 酒店=1, 火车=2, 门票=3)"
        }
      },
      "required": []
    }
  },
  {
    "method": "{category}.search",
    "description": "{品类名称}搜索",
    "category": "RESOURCE",
    "subCategory": "{CATEGORY}",
    "action": "SEARCH",
    "parameters": {
      "type": "object",
      "description": "{品类名称}搜索请求参数",
      "properties": {
        "cityId": {
          "type": "integer",
          "description": "城市ID",
          "example": 4
        }
        // ... 其他参数
      },
      "required": ["cityId"]
    }
  }
]
```

**关键字段说明**：

| 字段 | 说明 | 示例 |
|------|------|------|
| `method` | 接口方法名，用于调用 | `hotel.search`、`train.detail` |
| `description` | 接口描述 | `酒店搜索` |
| `category` | 一级分类 | `RESOURCE`、`ORDER`、`PASSENGER` |
| `subCategory` | 二级分类 | `HOTEL`、`TRAIN`、`FLIGHT`、`COMMON` |
| `action` | 动作类型 | `SEARCH`、`DETAIL`、`BOOK`、`REFUND` |
| `parameters.properties` | 参数定义 | 包含 type、description、example 等 |
| `parameters.required` | 必填参数列表 | `["cityId", "arrivalDate"]` |

**常用 action 类型**：

| action | 说明 | 适用场景 |
|--------|------|----------|
| `CITY` | 城市列表 | 资源类-通用 |
| `SEARCH` | 搜索/查询列表 | 资源类 |
| `DETAIL` | 详情 | 资源类 |
| `LIST` | 列表 | 乘客类 |
| `SAVE` | 保存 | 乘客类 |
| `DELETE` | 删除 | 乘客类 |
| `VALIDATE` | 验价 | 订单类 |
| `BOOK` | 预订/下单 | 订单类 |
| `QUERY` | 查询详情 | 订单类 |
| `STATUS` | 状态查询 | 订单类 |
| `CANCEL` | 取消 | 订单类 |
| `REFUND` | 退票/退订 | 订单类 |
| `HISTORY` | 历史列表 | 订单类 |
| `PAY` | 支付 | 订单类 |
| `DEDUCT` | 核损 | 订单类 |

## 必填字段收集原则（通用）

**不写死必填项**。在需要向后台发起请求时：
1. 查阅 [api/{category}.json](../api/{category}.json) 中对应 method 的 `parameters.required` 及 `properties`
2. 若用户未给出某必填字段，用自然语言提示用户填写（勿说「请提供某某参数」）
3. 每次只问一项缺项，待用户回复后再继续
4. 所有必填字段收集完整后再发起请求

**面向用户的话术**：对用户说的内容禁止出现「调用某某接口」「请求 API」等任何技术用语。

## 核心步骤与数据流（Agent 内部参考，勿对用户说）

```
1. 获取城市列表 → 得到目标城市信息
2. 查询{品类}列表 → 使用城市、日期等参数
3. 查询{品类}详情 → 获取具体资源信息
4. 查询乘客/入住人列表 → 若为空则先新增再继续
5. 创建订单 → 使用资源ID、乘客信息等
6. 轮询订单状态 → 预订完成后引导用户支付
```

## 完整交互流程

### 第一阶段：收集基本信息

**自然询问**，而非机械填表：

```
好的，我来帮您预订{品类}！
📍 在哪个城市？
📅 什么时候？（可以说"明天"、"本周六"、"3月15日"等）
```

**智能识别日期**：

| 用户输入 | 转换结果 |
|---------|---------|
| 今天/今日 | 当前日期 |
| 明天/明日 | 当前日期+1 |
| 后天、大后天 | 对应计算 |
| 下周一~下周日 | 计算对应日期 |
| 3月15日/3月15号/3-15 | 解析为 YYYY-MM-DD |

### 第N+1阶段：订单状态轮询（必执行）

- **轮询间隔**：10 秒
- **最大轮询次数**：6 次（最多等待约 1 分钟）

**状态码与处理**：

| 状态码 | 含义 | Agent 处理 |
|--------|------|-------------|
| `10` | 处理中 | 继续轮询 |
| `12` | 预订完成 | 展示成功反馈 |
| `11` | 预订失败 | 展示失败反馈 |
| 超时 | 轮询 6 次仍为 10 | 告知超时 |

### 步骤 5：后端接口映射（如需要）

如果后端需要在 `ApiActionEnum.java` 中添加新品类的映射：

```java
/**
 * 资源类--{品类}

RESOURCE_{CATEGORY}_SEARCH(CategoryType.RESOURCE.getCode(), SubCategoryType.{CATEGORY}.getCode(), ActionType.SEARCH.getCode(), {Category}Controller.class.getName(), "searchMethod", new String[]{SearchRequest.class.getName()}),

/**
 * 订单类--{品类}
 */
ORDER_{CATEGORY}_VALIDATE(CategoryType.ORDER.getCode(), SubCategoryType.{CATEGORY}.getCode(), ActionType.VALIDATE.getCode(), Single{Category}OrderController.class.getName(), "validateMethod", new String[]{ValidateRequest.class.getName()}),
ORDER_{CATEGORY}_BOOK(CategoryType.ORDER.getCode(), SubCategoryType.{CATEGORY}.getCode(), ActionType.BOOK.getCode(), Single{Category}OrderController.class.getName(), "createOrder", new String[]{CreateOrderReq.class.getName()}),

ORDER_{CATEGORY}_HISTORY(CategoryType.ORDER.getCode(), SubCategoryType.{CATEGORY}.getCode(), ActionType.HISTORY.getCode(), {Category}Controller.class.getName(), "orderList", new String[]{OrderListReq.class.getName()}),
```

**需要同步修改的文件**：

1. `SubCategoryType.java` - 添加新的二级分类枚举（如 `TICKET`）

3. `ApiActionEnum.java` - 添加接口映射
